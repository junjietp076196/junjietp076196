Library Management System

Python File Handling Console Application Team Project

[cite_start]A comprehensive console-based application designed to digitize library operations, managing interactions between librarians, students, and the book inventory[cite: 1].

Project Overview

[cite_start]This project was developed as a group assignment for the Python Programming module[cite: 1]. The objective was to create a robust system that replaces manual library record-keeping. [cite_start]The system distinguishes between two primary user roles: Librarians (Admin) and Students (Users), each with specific access privileges[cite: 1].

[cite_start]My primary contribution involved implementing the core logic for user authentication, administrative processing, and ensuring data persistence using text-based file handling[cite: 2].

Key Features

1. Role-Based Access Control

  • Librarian Portal: Secure login for admins to manage the system. [cite_start]Capabilities include adding new books, updating book details, removing obsolete records, and registering new users[cite: 1].
    • [cite_start]
  • Student Portal: User-friendly interface for students to search for books, view their borrowing history, and check the availability of specific titles[cite: 1].

2. Book Transaction Management

  • Borrowing System: Validates student eligibility and updates stock quantities in real-time.
    • [cite_start]
  • Returning & Fines: automatically calculates the duration of the loan and generates a fine amount if the book is returned overdue[cite: 1].

3. Data Persistence

    [cite_start]
  • The system utilizes file handling (`.txt` files) to store records permanently[cite: 1]. This ensures that user data and book inventories are saved even after the program terminates.
  • Implemented efficient reading/writing algorithms to handle data retrieval and updates.

Challenges & Learnings

One of the main challenges was ensuring data consistency when multiple operations (like borrowing and returning) modified the text files simultaneously. [cite_start]I learned valuable skills in defensive programming (validating user inputs to prevent crashes) and modular design by splitting the code into reusable functions for cleaner architecture[cite: 2].
