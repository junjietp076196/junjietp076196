APU Medical Centre (AMC) System

Java Swing GUI OOP Principles Text File DB

A Java-based desktop application designed to automate management processes for the APU Medical Centre, featuring role-based access for Admins and Doctors.

Project Overview

The APU Medical Centre (AMC) Management System was developed to address the inefficiencies of manual record-keeping. The system streamlines operations by allowing multiple users (Admin and Doctor) to interact with a centralized database of patient records, appointments, and medical inventory[cite: 4, 8].

The project strictly adheres to Object-Oriented Programming (OOP) principles, utilizing Inheritance, Encapsulation, and Polymorphism to create a modular and maintainable codebase[cite: 14].

Key Modules & Functionality

1. Administrator Module

The backbone of the system, allowing full control over clinic resources.

  • Staff Management: Capabilities to register new doctors, edit staff details, and manage user credentials.
  • Appointment Scheduling: View and manage patient appointments to ensure efficient time allocation.
  • Inventory Management: Track medicine stock levels and update supplies.

2. Doctor Module

A dedicated portal for medical professionals to manage patient care.

  • Schedule Management: Doctors can view their daily appointment slots and availability.
  • Consultation Records: Functionality to record patient symptoms, diagnosis, and issue prescriptions directly into the system[cite: 9].
  • Patient History: Access to past medical records for informed diagnosis.

3. Technical Implementation

  • Data Persistence: Implemented a robust text-file database system to store records (Appointments.txt, Patients.txt, etc.) persistently without an external SQL server[cite: 14].
  • Security: Secure login authentication system that verifies credentials against the stored user database before granting access[cite: 8].

Challenges & Solutions

A key challenge was implementing the file handling logic to ensure that data integrity was maintained when updating specific lines in the text files (e.g., modifying a single patient's record). I solved this by developing a helper class to read the entire file into an ArrayList, modify the specific object, and rewrite the file, ensuring no data was lost during the update process[cite: 14].
