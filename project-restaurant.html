APU Cafeteria Ordering System

C# Object-Oriented Programming System Architecture

A comprehensive management system designed to automate food ordering processes, minimize queues, and generate sales insights for the APU Cafeteria.

Project Overview

The APU Cafeteria Ordering System (ACOS) was developed to modernize the manual ordering process at the university cafeteria. The primary goal was to improve efficiency for both customers (students/staff) and cafeteria managers by digitizing menus, orders, and payment tracking.

Built using C#, the system demonstrates strong adherence to Object-Oriented Programming (OOP) principles, ensuring that the code is modular, reusable, and secure.

Key Modules & Functionality

1. Customer Module (Students/Lecturers)

Focused on ease of use and quick ordering.

  • Online Ordering: Customers can browse the digital menu, select items, and place orders without waiting in physical queues.
  • Profile Management: Users can update their personal details and manage their login credentials.
  • Order History: View past orders and current order status updates.

2. Manager Module (Admin)

Administrative control for cafeteria operations.

  • Menu Management: Full CRUD (Create, Read, Update, Delete) capabilities to update food items and prices based on availability.
  • Sales Reporting: Generate monthly income reports to track revenue and identify popular menu items.
  • Staff & Feedback Management: Register new staff members and view customer feedback to improve service quality.

3. Technical Architecture

  • OOP Implementation: Utilized Inheritance for user roles (extending a base 'User' class), Encapsulation for data security, and Polymorphism for handling different user behaviors.
  • Data Handling: Implemented robust exception handling to manage invalid inputs and ensure system stability during runtime.
